    crdt v0.1.0 API Reference

    Modules

    Crdt

    A library of Conflict-Free Replicated Data Types (CRDTs) written in pure Elixir

    Crdt.GCounter

    A G-Counter is a counter that only allows for increments

    Crdt.GSet

    A G-Set is a set where elements can only be added

    Crdt.LWWReg

    A LWW-Reg is a register that stores the value with the latest client provided timestamp

    Crdt.LWWSet

    A LWW-Set is set that supports removal and insertion of items an arbitrary number of times, but client must provide timestamps for these events. The set can be tuned in a way that it is biased towards adds or deletes, if an add operation occurs at the same timstamp of a remove operation

    Crdt.MVReg

    A MV-Reg is a register that maintains all values when there are conflicting entries. These conflicting entries are detected using Vector Clocks

    Crdt.ORSet

    An OR-Set Without Tombstones (ORSWOT) allows for insertions and removals of elements. Should an insertion and removal be concurrent, the insertion wins

    Crdt.PNCounter

    A PN-Counter consists of two Vector Clocks and allows for increments and decrements

    Crdt.TwoPSet

    A 2P-Set consists of two G-Sets and allows for an element to be added and removed at most once

    Crdt.VectorClock

    A Vector Clock is capable of generating a partial ordering of events in a distributed system and detecting causality vioations. It is essentially a map of actors to their logical clock value
